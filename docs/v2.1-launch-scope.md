## Post-launch (v2.2 and beyond)

These are planned but explicitly **out of scope** for the initial paid wave:

### Governance & lifecycle
- **Scopes & environments** (ii-030–033): Bind rules to repo/project/env; contextual MCP resolution.
- **Rule lint & CI** (ii-040–044): JSON/YAML lint pack, CLI exit codes, GitHub Action template.
- **Semantic search** (ii-050–053): Embeddings (local/BYO); duplicate-rule hints. Deferred pending embedding path decision.
- **Team workflow** (ii-060–062): Draft → review → approve states; review queue admin UI.

### Ops & trust
- **Telemetry/metrics** (ii-090–092): Aggregate counters (no rule body), metrics page, CSV export.
- **Secrets separation** (ii-095–097): Syntax for secret refs; env + Vault/Doppler resolvers.
- **Audit exports** (ii-100–101): NDJSON/JSONLines stream API; optional S3-compatible upload.

### Commercial packaging
- **Managed hosting tier** (ii-110): Positioning page + SLA checklist.
- **Air-gap bundle** (ii-111): Offline docs PDF, install checksums.
- **Enterprise SAML prep** (ii-112): Alignment with Business tier copy.

### Launch & growth
- **Positioning one-pager** (ii-120): vs MCP gateways vs IDE subscriptions.
- **Launch assets** (ii-121): Screenshot refresh, short demo script.
- **Support path** (ii-122): Issue template, security contact in README.
- **Post-launch feedback** (ii-123): First 5 deploying teams → triage into roadmap.

## Prioritisation logic

Features were selected for v2.1 must-have if they satisfy **at least two** of:

1. Required to complete a transaction (Stripe checkout, billing portal)
2. Required for an operator to run Instill safely in production (security, backups, TLS guide)
3. Required to make the versioning or webhook pitch credible (history UI, webhook dashboard, bundle import)
4. Creates a meaningful moat vs "just SQLite + MCP" (versioning, outbound webhooks, signed bundles)

Features that depend on external decisions (embedding provider, SAML IdP, Stripe live keys) or require significant product design (scopes, lint, team workflow) are post-launch.
