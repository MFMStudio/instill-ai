Per-client details

What to know before you connect.

🖱

Cursor

HTTP: add an MCP server block in .cursor/mcp.json pointing at your deployment with a Bearer token.

Stdio (advanced): open a checkout of the platform repo, use .env + dist/mcp-stdio.js for direct SQLite access — no web server required for MCP.

Open Setup Guide →
🤖

Claude Code

Merge the generated JSON into ~/.claude/.mcp.json or project .mcp.json. Use type: http and your Bearer API key.

Add the session-start instruction to CLAUDE.md so get_preferences runs automatically.

Open Setup Guide →
🖥

Claude Desktop

Configure claude_desktop_config.json (macOS / Windows / Linux paths in the guide) with the same URL and Authorization header.

Some builds prefer adding remote MCP under Settings → Connectors — use the same endpoint and token.

Open Setup Guide →
💙

VS Code · Copilot Chat

Use .vscode/mcp.json (schema varies slightly by VS Code version) with HTTP transport to /mcp.

Requires GitHub Copilot extension and a recent VS Code build with MCP support.

Open Setup Guide →
🌊

Windsurf

Edit ~/.codeium/windsurf/mcp_config.json and register the HTTP server with your Bearer key.

Open Setup Guide →
▶

Continue.dev

Add an HTTP MCP transport entry under experimental.modelContextProtocolServers in ~/.continue/config.json.

Open Setup Guide →
⚡

Zed

Merge context_servers in Zed's settings JSON; confirm field names against your Zed version's MCP docs.

Open Setup Guide →
💬

ChatGPT · Custom GPT

Not MCP — import /openapi.json as a Custom GPT Action. Use Bearer auth with your API key.

For OpenAI's servers to reach you, expose your host with a tunnel (e.g. ngrok) and use the HTTPS URL in the action definition.

Open Setup Guide →
