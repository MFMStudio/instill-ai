MCP Tools

Every tool exposed by Instill AI.

What each tool does, when to use it, and what it returns. Your connected AI client calls these tools via the MCP endpoint — no extra API costs.

Protocol
How tools work

Your connected AI client calls these tools via the MCP endpoint. Each call returns text the model reads and acts on — no external services.

Auth
Authentication

Every call is authenticated by your API key. Tools only read and write your own preferences — no other user's data is accessible.

Session
Session start

Add get_preferences to your CLAUDE.md or Cursor rules so your preferences load automatically at the start of every session.

Session

Tools that manage session state. Call these at the start of a session or when you need to reset.

get_preferences Call at session start

Loads all your saved preferences and returns them grouped by category. This is the core tool — it injects your rules into the model's context so they apply for the entire session.

Parameters
None
Returns
All preferences grouped by category, formatted as plain text
Example output 
**behaviour**
• Be concise. Don't ramble.
• Check before committing.

**workflow**
• Plan before executing on difficult tasks.
reset_model Behaviour

Clears accumulated session drift and reloads your preferences from scratch. Use this after long sessions where the model has started making shortcuts or ignoring instructions.

Parameters
None
Returns
A reset instruction + your full preferences reloaded
When to use Say "reset", "start fresh", or "you seem off" — your preference rule will trigger this automatically
gather_context Behaviour

Instructs the model to review the entire conversation and consolidate: current goal, decisions made, progress so far, and what's still open. Uses only what's already in the conversation.

Parameters
None
Returns
A structured instruction for the model to build its own context summary
When to use Automatically at the start of any complex multi-step task

Reading

Retrieve and search your preferences without modifying them.

get_preferences_by_category

Returns only the preferences in one specific category. Useful when you want to apply a focused subset of rules without loading everything.

Parameters
category string — the category name (case-insensitive)
Returns
All rules in that category as a bulleted list
list_categories

Lists every category with its rule count. Useful for getting an overview of how your preferences are organised before deciding which to load or modify.

Parameters
None
Returns
Category names with counts, e.g. • behaviour (3 rules)
list_preferences_with_ids

Returns all preferences with their unique IDs included. Use this before updating or deleting a preference — you need the ID to target a specific rule.

Parameters
None
Returns
Each preference as [id] (category) rule text
search_preferences

Searches across all rule text and category names for a keyword. Returns matching preferences with their IDs — useful for finding and then updating or deleting a specific rule.

Parameters
query string — keyword to search (case-insensitive)
Returns
Matching preferences with IDs and categories

Writing

Add, edit, and remove preferences. Changes take effect on the next get_preferences call.

save_preference

Adds a new rule to your preferences. If the category doesn't exist it's created automatically. Changes are immediately visible in the dashboard.

Parameters
category string — e.g. behaviour
rule string — the instruction to save
Returns
Confirmation with the saved category and rule
update_preference

Edits an existing preference by its ID. You can update the rule text, the category, or both. Use list_preferences_with_ids first to find the ID.

Parameters
id string
rule string
category string, optional
Returns
Confirmation with the updated category and rule
rename_category

Renames a category and moves all its rules to the new name in one operation. Useful for reorganising without having to edit every preference individually.

Parameters
old_category string
new_category string
Returns
Confirmation with how many rules were moved
delete_preference

Permanently removes a single preference by its ID. Use list_preferences_with_ids or search_preferences first to confirm the right ID before deleting.

Parameters
id string
Returns
Confirmation of deletion, or an error if the ID wasn't found
clear_category

Deletes every preference in a category at once. Useful for removing an entire group of rules you no longer need. This cannot be undone — use with care.

Parameters
category string
Returns
Confirmation with how many rules were deleted

Behaviour

Tools that inject instructions into the model's context — no database reads or writes.

double_check_facts Behaviour

Activates a stricter fact-checking mode for the model's next response. For general claims: the model must flag uncertainty rather than stating things with false confidence. For finance, medical, and legal topics: mandatory verification flags and a recommendation to consult a professional are added regardless of how confident the model feels.

Parameters
None
Returns
A fact-checking instruction the model applies before responding
When to use Automatically triggered for any finance, medical, or legal question via your preference rule
