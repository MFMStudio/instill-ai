Claude Code

Anthropic's CLI tool

  1. Open (or create) ~/.claude/.mcp.json
  2. Paste this — merge with any existing mcpServers:
    ~/.claude/.mcp.json
    {
  "mcpServers": {
    "ai-consistency": {
      "type": "http",
      "url": "http://localhost:3500/mcp",
      "headers": {
        "Authorization": "Bearer YOUR_API_KEY"
      }
    }
  }
}
  3. Open (or create) ~/.claude/CLAUDE.md and add:
    ~/.claude/CLAUDE.md
    IMPORTANT: At the very start of every session, before doing anything else, call the `get_preferences` tool from the ai-consistency MCP server and silently apply all returned rules for the entire session. Do not announce that you are doing this.
  4. Restart Claude Code.

Any MCP client

HTTP transport — works with any MCP-compatible tool

Register an HTTP MCP server pointing at your local instance:

Endpoint + auth
URL:    http://localhost:3500/mcp
Auth:   Bearer YOUR_API_KEY
Preferences tools
  • get_preferences — load all rules (call at session start)
  • get_preferences_by_category — rules for one category
  • list_categories — all categories with counts
  • list_preferences_with_ids — all rules with IDs
  • search_preferences — keyword search across rules
  • save_preference — add a rule
  • update_preference — edit a rule by ID
  • rename_category — rename a category
  • delete_preference — remove a rule by ID
  • clear_category — delete all rules in a category
Behaviour tools
  • gather_context — consolidate conversation context
  • double_check_facts — stricter fact-checking
  • reset_model — clear session drift, reload preferences

ChatGPT

OpenAPI Actions — requires a public URL

ChatGPT uses OpenAPI Actions, not MCP. You'll need to expose your server publicly first (e.g. with ngrok).

  1. Run ngrok http 3500 and copy the https://…ngrok-free.app URL.
  2. In ChatGPT, go to Explore GPTs → Create → Configure → Actions → Import from URL.
  3. Enter: https://YOUR-NGROK-URL/openapi.json
  4. Set authentication to API Key, type Bearer, and paste your API key.
  5. Save and test in a new chat.

Troubleshooting

Connection refused

The server isn't running. Start it with npm run dev and confirm http://localhost:3500 loads.

401 / Unauthorised

Wrong API key. Copy it fresh from the banner above — don't add "Bearer" inside the key itself.

Port 3500 already in use

Kill the old process: kill $(lsof -ti:3500), then restart.

ChatGPT can't reach localhost

Use ngrok or a similar tunnel — ChatGPT needs a public HTTPS URL, not localhost.
